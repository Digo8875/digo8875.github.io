Tutorial - Adding random quote to post with Jekyll Chirpy

Tutorial for implementing random quotes in blog posts

Posted Jan 14, 2025
By Rodrigo Candido
7 min read

Tutorial - Adding random quote to post with Jekyll Chirpy

Introduction

This article is a tutorial that provides an approach to configuring and creating quotes to be added to posts.
For the example we will use code that already exists on GitHub to continue with the implementation.
The idea is to bring more personality to our blog, through our favorite quotes. We will also work to make it work with our multilanguage according to the existing blog code from the previous tutorial: Tutorial - Creating a blog with Jekyll Chirpy.

Who is it for?

  • Developers who want to have quotes on their blog with Jekyll Chirpy

The problem

Implement a logic to add random quotes to our posts with each blog deployment, respecting our multilanguage functionality.
The implementation should allow us to configure our phrase, author and possibly an image for the quote.

Requirements

  • Tools
    • VSCode
    • Git
    • NodeJS
    • Ruby
    • Jekyll
  • What should i know?
    • Knowledge of VSCode or an IDE of your choice
    • Knowledge of GitHub

1 - The application used as a basis

Download the application that we will use as a base from GitHub, specifically this branch 7-github-io;
You can clone it using the Git command or download the zip and extract it to a folder for us to work on. Do as you wish;
With the application in hand, open it with VSCode or your preferred IDE and let’s get to work.

2 - The idea

Okay, let’s think about our problem and solution.
We have a blog that is already implemented with multilanguage and we want to put random quotes at the end of each post we make, according to each language.
So we will basically have to implement:

  • page with code to display the quote, which will be included at the end of our post;
  • styling the CSS for our quote;
  • random selection of the quote that will be displayed;
  • configuring the quotes, according to the language and image;
  • thinking about how our configuration and folders for images will be standardized;

With the idea in mind, let’s make our adjustments?

  {% assign quote_author = chosen_quote.[site.active_lang].author %}

{% if quote_image == null %}
  {% assign has_image = false %}

<div class="quote-wrapper">

    <div class="quote-phrase-decoration">

    </div>

    <div class="quote-phrase">
      {{ quote_phrase }}
    </div>
  </div>

  {% if quote_image == null %}
    <div class="quote-division"></div>
  {% endif %}

  {% if quote_author %}
    <div class="quote-author">
      - {{ quote_author }}
    </div>
  {% endif %}

</div>

This is our implementation for displaying quotes and by default the file will be in the /_includes folder, as it will always be added somewhere in another Frontend file.
In the first line we will be taking all the quotes from our configuration and selecting one to be added to the post.
Next, our code will initialize variables according to our default language, and then analyze the settings for the current language so we can continue with the logic.
In the DIV quote-wrapper we have the implementation of how we want the display on the Frontend and as you can see, we have the use of our configured variables for phrase, author and image.

With our display created, let’s add where it will be included to be displayed at the end of our posts.

Since we created a new style file, we will need to adjust it so that it is added together with the other SASS files when the application is compiled.
Go to the file /_sass/layout/_index.scss and add the import of our new file, so that it looks like this:

Let’s now adjust the styling for the application’s light and dark themes.
In the /_sass/themes/_light.scss file, add the variable configuration:

2

/* Quotes */
--quote-text-shadow: rgb(175, 176, 177);

Do the same for the dark theme in the /_sass/themes/_dark.scss file and its variable:

1
2

/* Quotes */
--quote-text-shadow: #151515;

This is the styling I implemented, feel free to style it as you wish later and also change more variables for your light and dark themes.

As you can see in our configuration files, we can:

  • create citations with multiple languages ​​or just the default language;
  • include or not an image in the citation;
  • use HTML tags to place the citation on multiple lines;

Now all that’s missing is our display image, right?
I copied our avatar image as an example, from /assets/img/site to our new quotes folder /assets/img/quotes/rodrigo–my_phrase_1.png
I chose to follow a pattern for my images like: author–title_phrase, feel free to follow your standardization, since the images for each post will be in your configuration in /_data/quotes.yml

Let’s test it?
Run the application with bundle exec jekyll s and access a post.
Remember that the quotes will be random and assembled when the application is compiled, so to see the quotes changing it will be necessary to change something in the code for the server to restart or stop and start the application again.
